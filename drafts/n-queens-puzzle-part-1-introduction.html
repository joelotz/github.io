N-Queens Puzzle, Part 1-Introduction

 June 13, 2020      Stardate:     Tagged as: Python

The N-Queens puzzle is the popular problem of placing N number of chess queens on an NxN grid board such that no queens cannot attack each other; i.e no two queens share the same row, column, or diagonal. The puzzle is often generalized to the “8 Queens Puzzle” to match a standard size chess board and is a good example of a simple but nontrivial algorithm problem.

It is known that a solution exists for all N>3 It gets very difficult to do by hand for N>7 Good candidate for computer (programmatic) solution

Encoding

In the example solution below, a 4x4 board has 4 queens placed on it, denoted by red X’s. The indices are zero indexed, meaning they start at zero. For a generalized NxN grid(aka. board) the indices go from 0 to N-1. For the time being, it doesn’t matter where the queens are placed as we will be learning some background needed.

One way of encoding the queen locations are to use a list of ordered pairs representing the row and column positions.

$$[ (row_0,col_0), (row_1,col_1), (row_{N-1},…,col_{N-1}) ]$$

The above example can be encoded as $[ (1,0),(3,1),(0,2),(2,3)]$.

We can reduce the dimensionality by recognizing the column value is already represented by the location within the list.

Therefore, the example can be encoded as $[1,3,0,2]$.

Combinations and Permutations

Combinations and permutations are often confused; for our purposes I will simply say that one cares about order while the other does not, and this makes a difference. In combinations the pair $(0,1)$ is the same as $(1,0)$. This is good for relationships or interactions because the interaction between John and Bill is the same and between Bill and John, we don’t care about order.

On the other hand, let’s say… for example… I want to calculate all the possible locations of queens without duplication of rows. $$ P_N = N! $$ $$ P_4 = 4!= 4 \cdot 3 \cdot 2 \cdot 1 = 24$$

Finally, the Brute-Force Algorithm

The Brute-Force Algorithm involves finding all possible combination of queen positions and then evaluating each to determine if it is a valid solution.

Relationships = Combinations

Now that we have defined our queen placement as an ordered list, $Queens=[1,3,0,2]$, we can discuss the interaction or relationship between each of them. Mathematically these are called combinations. Given a collection of queens, we can show each possible interaction between any two queens as $\binom {Queens}k$.

The combination $A=(0,1)$ is the interaction between the queen in row 0 and the queen in row 1, as so on. $A = (0,1)$ $B = (0,2)$ $C = (0,3)$ $D = (1,2)$ $E = (1,3)$ $F = (2,3) $
